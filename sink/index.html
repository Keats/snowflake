Grid

The grid is based on flexbox (see this link for more info: a guide to flexbox) and allows for some funky things compared to normal grids.
This grid is inspired (and for some parts copied) on the one from Foundation Apps.
Unlike the other frameworks, this doesn't come with predefined classes, it only contains mixins.

Grid frame

This is the outermost container for your app. It takes 100% of the height and width of the browser. The mixin is called grid-frame.

Grid blocks

Those are flexbox elements that can do an awful lot of things. The mixin is called grid-block. Let's see some example of what it can do.

Automatic sizing

Blocks take an even amount of space.

Manual sizing

You can ensure a block only takes the amount of space it needs by using the grid-fit mixin. In the example below, the left block is 42px long.

It also doesn't contain classes for x columns like most framework, you can choose a length by requesting a percentage using the mixin grid-size. The left block takes 33% in the example belox.

Vertical layout

Flexbox works vertically as well as horizontally. This is an option to the grid-block mixin.

Container

By default the block will take all available space. grid-container is there in case you want to wrap a block with a max width.

Source ordering

Flexbox allows to change the order of an element. Use the grid-order mixin for that, a lower number meaning they appear first.

Typography

Headers

Headers are based major third modular scale with a base fontsize of 16px. Check http://type-scale.com/ to make your own. h5 and h6 don't have specific style as they should not really be used.

Paragraphs

This is a paragraph. It comes with a set line height/size matching the rest of the HTML elements. Use em to put italicize words, strong to bolden them and small to write smaller text.
